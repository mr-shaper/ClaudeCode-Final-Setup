# Claude Code 本地化/自定义 API 完美解决方案
# (Claude Code Custom API Solution)

## 📖 这是什么？(What is this?)
这是一个让你能用 **Kimi、Opendoor 或其他 OpenAI 格式的 API** 来驱动 **Claude Code**（Anthropic 官方强大的编程助手）的完整解决方案。

通常情况下，Claude Code 只能用官方昂贵的 Anthropic 账号。但通过这个方案，你可以用自己便宜、方便的 API 来让 Claude 帮你写代码、操作电脑。

---

## 💡 原理是啥？(How it works?)
想象一下：
*   **Claude Code (厨师)**：它只懂法语（Anthropic 协议），手艺很好，能帮你做大餐（写代码、改文件）。
*   **你的 API (供应商)**：比如 Kimi，它只懂中文（OpenAI 协议），食材便宜又新鲜。

**问题**：厨师听不懂供应商的话，没法下单买菜，所以罢工了。

**本项目的解决方案**：我们请了一个 **"翻译官" (Proxy)** 站在中间。
1.  厨师（Claude）把需求告诉翻译官。
2.  翻译官把需求翻译成中文告诉供应商（API）。
3.  供应商把食材给翻译官，翻译官再给厨师。
4.  **结果**：厨师开心地做好了大餐（成功创建文件、执行命令）。

### ✨ 独家黑科技：文档解析增强
普通的 Proxy 只能传文本，但我们给这个 Proxy 装上了 **"超级阅读器"**！
*   ✅ **PDF / Word / PPT**：自动提取文本内容。
*   ✅ **Excel**：自动提取表格数据。
*   这意味着你可以直接把这些文件扔给 Claude，让它帮你总结、分析数据！

## 🚀 怎么开始？(Quick Start)

1.  打开 **`User_Manual.md`**。
2.  按照里面的 **"首次安装 / 一键还原"** 章节操作。

5.  **开始享受 Claude Code 的强大能力！**

## ⚠️ 常见问题

*   **为什么要填两个 Key？**
    *   一个给翻译官（Proxy）用来真正买菜。
    *   一个给厨师（Claude）用来检查"你有没有带钱"（虽然他不用，但他要检查格式）。所以两个填一样的就行。

*   **为什么我的 Claude 以前只会说话，不会写文件？**
    *   因为没有这个"翻译官"，厨师以为供应商没货，所以只能动嘴皮子。现在有了翻译官，他就可以动手了！
